# AuditPilot 规则与信息源主文档

## 1. 文档说明

本文档是当前版本的主维护文档，用于说明：

- 当前已落地的风险判定规则
- 当前正式运行态使用的信息源
- 当前系统依赖的核心特征、证据类型和知识增强来源

维护原则：

- 以后若调整规则、provider、信息源、主要特征工程口径或 LLM/RAG 来源说明，应优先更新本文档。
- 本文档以当前代码现状为准，不把未落地能力写成已实现能力。

## 2. 当前判定规则总览

### 收入确认风险

- 四季度收入占比异常偏高
- 应收账款增速显著高于营收增速
- 经营现金流与利润背离

### 存货减值与积压风险

- 存货增速高于营收增速且周转下降
- 行业需求下行时存货仍上升

### 应收账款回款风险

- 应收账款周转率下降且规模持续上升

### 合规与外部事件风险

- 重大诉讼、处罚、负面事件触发风险预警

### 关联交易与内控风险

- 关联结构复杂
- 高管频繁变动

## 3. 规则明细

### REV_Q4_SPIKE

- 规则名称：收入确认风险：四季度收入占比异常偏高
- 风险类别：财务风险
- 风险等级：HIGH
- 判定指标：`q4_revenue_ratio`
- 阈值 / 逻辑：`all`，四季度收入占比 `> 0.35`
- 重点科目：主营业务收入、应收账款、合同资产
- 重点流程：收入确认、期末截止、订单履约
- 建议程序：截止测试、抽查大额合同与签收单、检查期后退货与折让
- 建议证据：销售合同、出库与签收单、期后回款记录
- 当前实现状态：已实现

### REV_AR_GAP

- 规则名称：收入确认风险：应收账款增速显著高于营收增速
- 风险类别：财务风险
- 风险等级：HIGH
- 判定指标：`ar_revenue_growth_gap`
- 阈值 / 逻辑：`all`，差值 `> 0.1`
- 重点科目：应收账款、坏账准备、主营业务收入
- 重点流程：授信审批、销售回款、收入确认
- 建议程序：函证、账龄分析、期后回款测试
- 建议证据：应收账款明细、账龄结构、客户回款流水
- 当前实现状态：已实现

### OCF_PROFIT_DIVERGENCE

- 规则名称：收入确认风险：经营现金流与利润背离
- 风险类别：财务风险
- 风险等级：HIGH
- 判定指标：`operating_cf_profit_ratio`
- 阈值 / 逻辑：`all`，经营现金流 / 净利润 `< 0.5`
- 重点科目：经营活动现金流量净额、主营业务收入、应收账款
- 重点流程：收款循环、收入确认、现金流分析
- 建议程序：比较利润与现金流差异、复核重大应收项目、抽查异常订单
- 建议证据：现金流量表、银行回单、客户对账资料
- 当前实现状态：已实现

### INV_BACKLOG

- 规则名称：存货减值 / 积压风险：存货增速高于营收增速且周转下降
- 风险类别：经营风险
- 风险等级：HIGH
- 判定指标：`inventory_revenue_growth_gap`、`inventory_turnover_delta`
- 阈值 / 逻辑：`all`，前者 `> 0.1` 且后者 `< -0.2`
- 重点科目：存货、存货跌价准备
- 重点流程：采购与生产计划、仓储管理、减值测试
- 建议程序：存货监盘、库龄分析、跌价准备复核
- 建议证据：库存台账、库龄清单、减值测算表
- 当前实现状态：已实现

### INV_INDUSTRY_DOWN

- 规则名称：存货减值 / 积压风险：行业需求下行时存货上升
- 风险类别：经营风险
- 风险等级：MEDIUM
- 判定指标：`industry_demand_down_inventory_up`
- 阈值 / 逻辑：`all`，行业下行背景下存货仍上升
- 重点科目：存货、营业成本
- 重点流程：销售预测、生产排产、库存管理
- 建议程序：对比行业景气与企业产销数据、复核滞销清单
- 建议证据：行业数据、产销报表、库存结构分析
- 当前实现状态：已实现

### AR_COLLECTION

- 规则名称：应收账款回款风险：回款周期延长
- 风险类别：财务风险
- 风险等级：HIGH
- 判定指标：`ar_turnover_delta`、`accounts_receivable_growth_rate`
- 阈值 / 逻辑：`all`，应收周转率下降且应收规模明显上升
- 重点科目：应收账款、坏账准备
- 重点流程：信用管理、回款管理
- 建议程序：检查账龄结构、执行大额客户回款测试、复核坏账准备
- 建议证据：账龄表、回款台账、坏账测算底稿
- 当前实现状态：已实现

### COMPLIANCE_EVENTS

- 规则名称：合规与外部事件风险：诉讼处罚及负面事件
- 风险类别：合规风险
- 风险等级：MEDIUM
- 判定指标：`major_litigation_count`、`penalty_count`、`negative_sentiment_count`
- 阈值 / 逻辑：`any`，任一项存在即触发
- 重点科目：预计负债、营业外支出、披露相关科目
- 重点流程：合规管理、法务审查、信息披露
- 建议程序：获取处罚资料、访谈法务与合规负责人、复核披露充分性
- 建议证据：处罚决定书、法律函件、董事会纪要
- 当前实现状态：已实现

### RELATED_PARTY_CONTROL

- 规则名称：关联交易 / 内控风险：关联结构复杂且高管频繁变动
- 风险类别：内控风险
- 风险等级：MEDIUM
- 判定指标：`related_party_complexity_score`、`executive_change_count`
- 阈值 / 逻辑：`any`，任一项异常即可触发
- 重点科目：关联交易、其他应收应付、采购成本
- 重点流程：关联方识别、审批授权、内控执行
- 建议程序：获取关联方清单、检查审批链条、复核异常交易定价
- 建议证据：关联方台账、审批记录、合同与定价依据
- 当前实现状态：已实现

## 4. 当前信息源与渠道总览

### 财务与结构化指标

- 当前正式来源：AkShare
- 当前调用方式：企业主数据与财务指标同步
- 当前使用状态：已实现
- 进入系统的模块：企业主数据、财务指标、风险计算
- 典型用途：企业引入、基础财务字段、风险分析前置数据

### 公告与审计报告

- 当前正式来源：巨潮资讯
- 当前调用方式：`sync/company` 同步公告、年报、审计报告、处罚公告
- 当前使用状态：已实现
- 进入系统的模块：审计概览、时间线、文档中心、证据链
- 典型用途：官方文档入库、处罚事件入库、问答引用

### 上传文档

- 当前正式来源：用户上传
- 当前调用方式：文档上传、解析、抽取
- 当前使用状态：已实现
- 进入系统的模块：文档中心、知识库、问答引用
- 典型用途：补充企业年报、公告、底稿和证据材料

### AI / LLM / RAG

- 当前正式来源：MiniMax + 本地轻量 RAG
- 当前调用方式：风险解释、问答生成、知识块检索
- 当前使用状态：已实现
- 进入系统的模块：风险解释、问答页、建议生成
- 典型用途：解释规则结果、组织证据、回答用户问题

## 6. 风险判定所依赖的特征与证据类型

### 当前主要特征

- 营收增长率
- 净利润增长率
- 经营现金流 / 净利润
- 存货增长率
- 应收账款增长率
- 存货周转率变化
- 应收账款周转率变化
- 毛利率波动
- 资产负债率变化
- 期间费用率变化
- Q4 收入占比
- 行业需求下行且存货上升

### 当前主要证据类型

- 财务指标
- 公告
- 年报 / 审计报告
- 处罚 / 监管事件
- 上传文档
- 结构化风险结果

## 7. 已知限制与后续补充方向

### 当前限制

- `AkShare` 当前只承担企业主数据和结构化数据，不承担公告主源职责。
- 公告、年报、审计报告、处罚事件当前主要依赖巨潮资讯。
- 轻量 RAG 可用于正式问答，但仍不是生产级知识库检索系统。
- 商业数据库、工商直连、全面监管直连仍未纳入正式运行态。

### 后续方向

- 扩展更稳定的外部监管与工商数据源
- 提升文档解析准确率和知识块质量
- 优化规则权重与历史样本校准
